P
Prompt 框架库 · 使用指南
5 分钟上手，终身受益
◈
这是什么？
一个运行在本地浏览器的 Prompt 框架管理工具。帮你把每次调教 AI 的好 Prompt 沉淀下来，形成你专属的「提问方法论」。
◇
解决什么问题？
每次用 AI 都要重新想怎么提问？好的 Prompt 用完就丢了？这里帮你存、用、迭代，越用越好。
核心功能一览
库
Prompt 框架库
按场景分类存储，支持标签、搜索、质量层级标注，随时取用。
磨
AI 打磨（需配置 API）
描述你的需求，AI 帮你生成并多轮迭代优化，每轮可对比新旧版本。
则
原则库
记录你总结的 Prompt 规律，新增时随时参考，积累个人方法论。
评
使用反馈
每次复制后评分（好用 / 一般 / 需改进），追踪哪些框架真正有效。
份
导入 / 导出
数据存本地浏览器，可随时导出 JSON 备份，换设备时导入恢复。
推荐的完整使用路径，从零开始到建立自己的方法论。
01
配置 API （可选，用于 AI 打磨）
侧边栏底部「API 设置」→ 选择服务商 → 填入 Key → 连接测试。不配置也可以手动编写 Prompt。
02
新增你的第一条 Prompt
点右上角「+ 新增」，在 AI 打磨区输入需求，或直接在内容框手写框架。选择场景、打上标签、标注质量层级。
03
使用并评分
点卡片打开详情 → 点「复制 Prompt」→ 粘贴到 AI 工具使用 → 回来给效果评分（好用 / 一般 / 需改进）。
04
迭代改进
对「需改进」的 Prompt，点编辑 → 在 AI 打磨区提出修改意见 → 对比新旧版本 → 保存最佳版本。
★
沉淀原则
用了一段时间后，把反复有效的技巧记录到「原则库」，下次写 Prompt 时一键追加，形成你的专属方法论。
本库使用「五要素框架」组织所有 Prompt。不是每条都要包含全部五项，而是按需取舍——要素越完整，AI 的输出越可预测。
①
角色设定 Role
给 AI 赋予一个专业身份。身份越具体，输出越专业。
「你是一位拥有 15 年经验的价值投资分析师，深受巴菲特哲学影响」
②
背景约束 Context
提供情境、受众、限制条件。让 AI 知道「这是什么场景」。
「背景：我需要向董事会汇报，受众是非技术背景的高管，时间限制 10 分钟」
③
任务指令 Task
明确要做什么，最好用编号列出步骤。越具体越好。
「任务：1. 分析核心风险 2. 提出3个解决方案 3. 给出推荐理由」
④
输出格式 Format
指定结构、长度、语气。让输出结果开箱即用。
「输出格式：Markdown，含小标题，500 字以内，专业严谨语气，结尾附行动建议」
⑤
质量标准 Quality
告诉 AI「什么是好的输出」，这是最容易被忽略、也最有效的要素。
「质量标准：读完后受众能立刻理解下一步行动；风险分析篇幅不少于优势分析」
AI 打磨功能需要先在「API 设置」里配置好接口。配置后，你只需描述需求，AI 自动帮你生成并多轮打磨。
STEP 1
描述你的场景 — 不用完整，用粗糙语言描述「我想用 AI 做什么」就够了。
STEP 2
点「生成 / 再迭代」 — AI 会按五要素框架生成第一版，自动填入内容框。
STEP 3
对比 + 提出修改意见 — 第一版出来后，点「提出修改意见」，描述你觉得哪里不够好。
STEP 4
再次迭代 — 每轮都能看到新旧版本对比，选择「使用新版本」或「保留旧版本」。

技巧 01
质量标准写「行为结果」，不写「形容词」
✗ 差：「高质量、专业、详细」 ✓ 好：「读完后受众能立刻知道下一步做什么」
技巧 02
角色设定加年限或背景，效果立竿见影
✗ 差：「你是一位分析师」 ✓ 好：「你是一位拥有 12 年 A 股研究经验的卖方首席分析师」
技巧 03
分析类任务必加「区分事实与推断」
在任务指令末尾加上这一句，能显著提升分析类输出的可信度，减少 AI 编造数据。
技巧 04
用 [ ] 标注变量，让框架可复用
把每次变化的部分用方括号标注：[公司名称]、[目标受众]。这样同一个框架可以重复用于不同场景。
技巧 05
效果不好？先看是哪个要素出了问题
输出太泛 → 检查「任务指令」是否够具体
格式乱 → 检查「输出格式」是否明确
内容不专业 → 检查「角色设定」是否够细
验收困难 → 补充「质量标准」
